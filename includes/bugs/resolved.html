I cannot download and view a PDF

Date Reported

August 2011

Problem Description

Users cannot display and/or download a PDF. When the PDF link is clicked, an empty window or a download plugin message appears. This is because PDFs that were uploaded using the "File" content type in Blackboard are not compatible with all browsers and versions and there is no one solution that will work for all browser types and for some browser versions there is no solution to the problem.

Resolution Status

Resolved

Workaround

N/A

Student cannot Submit a Test/Assessment

Date Reported

June 2011

Problem Description

Students taking a test will submit their answers and receive an error message directing them to refresh their screen. After refreshing, Blackboard will prompt the user to access the exam and submit again. Upon attempting this second submit, a message will appear explaining that the user no longer has access to the exam as it has already been completed.

Resolution Status

Resolved

Workaround

Students should save each answer as they go. When students have saved their answers, instructors can go into the Grade Center, view the attempts, and submit on behalf of the students. If there is not any work there (likely because students were not saving after each answer) it is necessary for instructors to clear the original attempt and students will need to take the test/assessment again.

Lost Work Because the Browser Times Out

Date Reported

June 2011

Problem Description

Instructors working in Blackboard for an extended period of time get logged out or receive an error message causing them to lose work. A generic Blackboard error message appears (e.g. "multipart/form-data not allowed").

Resolution Status

Resolved

Workaround

Consider saving work often and when performing tasks that are time consuming (as when using the Grade Center), save frquently, and then refresh the browser window.
